Preface

Thank you for buying SocketIt - the connection library. SocketIt helps you to connect and group things together by providing premade components. You can use it to connect structures like LEGO blocks, body limbs, car parts or parts of buildings. It’s also helpful for creating slotted interfaces with snapping points or just to connect GameObjects virtually together so that they are able to communicate with each other.

Components

SocketIt is split into multiple drop-In components and can be configured to your needs. It makes heavy use of the Unity event system and is easily extendable. You can inject your code in almost every situation

Editor

SocketIt is fully integrated into the Unity editor with custom windows, shortcuts and gizmos. Every feature is usable ingame and inside the editor.

State

All properties are public without any hidden state. You can access, manipualte and serialize them.

Be aware that SocketIt is not a “finished game”. You will have to write code so that it integrates into your game.

Installation

Copy the Editor folder from /SocketIt/Assets/Editor into your applications root directory. This step is neccessary for Unity to activate the editor features.

Concept

In SocketIt modules can be connected with their sockets. A module can have multiple sockets. Imagine a LEGO brick. You can connect the brick in various combinations but always limited to the nobs and holes of the brick. A socket in SocketIt represents both the knobs and holes.

[img gif of sockets]

Connected modules form a composition. A composition knows everything about its modules and can be used to propagate events and information to them.

Components

SocketIt consists mainly of 3 basic components: Compositions, Modules and Sockets.

Module

Modules form the basis blocks of socketIt. Every GameObject that should be connectable to others needs the Module Component. A Module by itself is kinda useless, it needs at least one socket component.

The normal structure of your modules looks like this:

A Module needs to be aware of its Sockets. You have to add them to the Sockets property of the Module

If you create a socket on a child element of a module, socketIt will recognize it and adds the socket automatically to the modules socket property.

If two Modules are connected, they form a Composition. A Module can only be part of one Composition at a time. Even though a Module needs at least one Socket to work properly, you are free to combine the Socket and Module component in one GameObject

Module Events

Event Description
On Snap Fired whenever a Socket of another Module snaps to a Socket ot this Module.
On Module Connected Fired whenever a Module connects to another Module.
On Module Disconnected Fired whenever a Module gets disconnected from a connected Module.
On Composition Changed Fired whenever a Module gets separated from its Composition or when a Composition gets split in half and the Module is part of the new Composition

All Module events are mirrored to the other Module. When two Module get connected, both will fire the On Module Connect event. See the Connection section for detailed information.

Socket

Sockets are the connection points of modules. A module must have at least one socket. Each socket has to know about its module within its Module property

If you add a socket component under a GameObject with a module component, the Module property of the Socket will be automatically filled

Sockets are the connection points of modules.

Socket Events

Event Type Description
On Connect SocketEvent Fired when two sockets are connected.
On Disconnect SocketEvent Fired when two sockets are disconnected.
On Snap SnapEvent Fired when two sockets snap together. See snapping for a detailed explanation
